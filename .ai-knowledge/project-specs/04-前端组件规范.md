# 前端组件规范

## 组件注册

### 全局组件
- 使用 `unplugin-vue-components` 自动注册 Element Plus 组件
- 自定义全局组件在 `components.d.ts` 中声明

### 局部组件
- 在 `<script setup>` 中直接导入使用
- 不需要手动注册组件

## ProTable 组件

### 基本用法
ProTable 是项目中用于快速构建 CRUD 页面的核心组件，集成了搜索、表格、分页、列设置功能。

```vue
<template>
  <div class="page-container">
    <ProTable
      ref="proTableRef"
      :title="t('page.title')"
      :columns="columns"
      :search-fields="searchFields"
      :data="dataList"
      :loading="loading"
      :show-pagination="true"
      :pagination="pagination"
      :search-params="searchParams"
      column-settings-key="unique_key"
      :show-selection="true"
      @query="onQuery"
      @selection-change="handleSelectionChange"
    >
      <template #header-buttons>
        <el-button type="primary" :icon="Plus" @click="handleOpenCreate">
          {{ t('common.add') }}
        </el-button>
      </template>

      <template #column-xxx="{ row }">
        <span>{{ row.xxx }}</span>
      </template>

      <template #column-actions="{ row }">
        <el-button type="primary" link size="small" :icon="Edit" @click="handleOpenEdit(row)">
          {{ t('common.edit') }}
        </el-button>
        <el-button type="danger" link size="small" :icon="Delete" @click="handleDelete(row)">
          {{ t('common.delete') }}
        </el-button>
      </template>
    </ProTable>
  </div>
</template>
```

### Props 说明
- `title` - 页面标题
- `columns` - 表格列配置数组
- `search-fields` - 搜索字段配置数组
- `data` - 表格数据
- `loading` - 加载状态
- `show-pagination` - 是否显示分页
- `pagination` - 分页参数 `{ page, pageSize, total }`
- `search-params` - 搜索参数对象
- `column-settings-key` - 列设置本地存储 key
- `show-selection` - 是否显示复选框

### 插槽
- `header-buttons` - 表格顶部操作按钮
- `column-{key}` - 自定义列渲染
- `column-actions` - 行操作按钮

## 样式规范

### 页面容器
- 使用 `class="page-container"` 作为顶层容器
- 页面背景色由全局样式控制

### 表格操作列
```css
.table-actions {
  white-space: normal;
  display: flex;
  gap: 4px;
  flex-wrap: wrap;
  align-items: center;
}
```

### 自定义颜色
- 使用 CSS 变量 `var(--mainColor)` 作为主题色
- 使用 Element Plus 的 `el-tag` 类型代替自定义颜色

## 状态管理

### 使用 Pinia
```typescript
// stores/xxxStore.ts
import { defineStore } from 'pinia'

export const useXxxStore = defineStore('xxx', () => {
  const data = ref([])
  const loading = ref(false)

  const fetchData = async () => {
    loading.value = true
    try {
      data.value = await xxxApi.list()
    } finally {
      loading.value = false
    }
  }

  return { data, loading, fetchData }
})
```

## 路由

### 动态路由
- 使用 `src/router/dynamicRoutes.ts` 定义动态路由
- 路由路径使用 kebab-case
- 路由名称使用 PascalCase

### 路由守卫
- 未登录跳转登录页
- 无权限显示 403 页面
- 路由切换显示进度条（NProgress）

## 权限控制

### 指令方式
```vue
<el-button v-permission="['user:create']">新建</el-button>
```

### 组件方式
```vue
<Permission :permissions="['user:create']">
  <el-button>新建</el-button>
</Permission>
```

## 错误处理

### 全局错误处理
- 使用 `src/utils/error.ts` 统一处理请求错误
- 500 错误显示"服务器错误"
- 401 错误自动跳转登录页
- 403 错误显示"无权限"

### 组件级错误
- 使用 `ErrorBoundary.vue` 包裹可能出错的组件
- 使用 `try/catch` 处理异步操作错误
