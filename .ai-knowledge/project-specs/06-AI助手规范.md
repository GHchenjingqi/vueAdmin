# AI 助手规范

> AI 代码生成助手是项目的正式功能模块，已接入前后端路由。

---

## 一、前端组件（src/components/AIAssistant/index.vue）

### 1.1 组件结构

- **浮动触发按钮**：右下角圆形按钮（52px），渐变背景，点击展开抽屉
- **抽屉式对话界面**：右侧抽屉（540px），`el-drawer` 实现
- **消息列表**：用户/AI 气泡式对话，支持 Markdown + 代码高亮
- **代码文件预览**：文件列表展开、代码复制、单文件/批量应用
- **AI 提供商管理弹窗**：内嵌在抽屉中，支持增删改查、启用/禁用切换
- **模型/提供商切换**：下拉菜单（`el-dropdown`），支持多提供商多模型
- **Token 用量展示**：每条 AI 回复下方显示 Token 统计

### 1.2 代码高亮

- `highlight.js` 使用**懒加载**（`import()` 动态导入），避免拉入首屏
- 高亮主题：暗色（VS Code 风格），覆盖 `.ai-code-viewer .hljs-*`
- 使用 `hljsReady.value` 响应式变量触发重渲染（hljs 加载完成后重新高亮）

### 1.3 Markdown 渲染

- 使用 `marked` 库（GFM 模式）渲染 AI 回复
- 配置：`marked.setOptions({ breaks: true, gfm: true })`
- 支持 Mermaid 图表和 KaTeX 公式（通过 `md-editor-v3` 集成）

### 1.4 快捷键

- `Enter`：发送消息
- `Shift + Enter`：换行
- `Ctrl + Enter`：发送消息

### 1.5 AI 提供商管理

- 内嵌 `el-dialog` 弹窗，包含提供商列表和编辑表单
- 提供商列表：显示名称、API URL、模型标签、启用/禁用开关
- 编辑表单：名称、API URL、API Key（密码模式，编辑时留空不修改）、模型列表、默认模型

## 二、API 调用（src/api/ai.ts）

### 2.1 接口定义

```typescript
export const aiApi = {
  // 发送聊天消息
  chat: (data: { message: string; providerId?: number; model?: string }) =>
    request.post<{ explanation: string; files?: GeneratedCodeFile[]; usage?: Usage }>('/ai/chat', data),

  // 应用生成的代码文件
  apply: (files: GeneratedCodeFile[]) =>
    request.post<{ message: string; createdMenuIds?: number[] }>('/ai/apply', { files }),

  // 获取 AI 服务状态
  getStatus: () => request.get<AIStatus>('/ai/status'),

  // 获取已启用提供商列表
  getEnabledProviders: () => request.get<AiProvider[]>('/ai/providers'),

  // 获取全部提供商列表
  getProviders: () => request.get<AiProvider[]>('/ai/providers/all'),

  // 创建 AI 提供商
  createProvider: (data: Partial<AiProvider>) => request.post('/ai/providers', data),

  // 更新 AI 提供商
  updateProvider: (id: number, data: Partial<AiProvider>) => request.put(`/ai/providers/${id}`, data),

  // 删除 AI 提供商
  deleteProvider: (id: number) => request.delete(`/ai/providers/${id}`),

  // 获取单个提供商详情（含 apiKey）
  getProvider: (id: number, withKey?: boolean) =>
    request.get<AiProvider>(`/ai/providers/${id}`, { params: { withKey } }),
}
```

### 2.2 关键类型

```typescript
interface GeneratedCodeFile {
  path: string             // 文件路径
  content: string          // 文件内容
  language: string         // 编程语言
  isNew: boolean           // 是否新文件
  description?: string     // 文件描述
}

interface AiProvider {
  id: number
  name: string
  apiBaseUrl: string
  apiKey?: string          // 敏感字段，仅 withKey=true 时返回
  models: string           // 模型列表，逗号分隔
  defaultModel: string
  enabled: number          // 1=启用, 0=禁用
}

interface AIStatus {
  apiConfigured: boolean
  message: string
  knowledgeBase: {
    totalFiles: number
    categories: Record<string, number>
    lastUpdated: string
  }
}
```

## 三、后端服务（server/services/）

### 3.1 AIAssistant.ts

- 多提供商大模型调用（通过 `AiProviderService` 管理）
- 结合本地 RAG 知识库检索（`LocalFileRAG`）
- 根据自然语言需求生成符合本项目规范的全栈代码
- 输出格式：JSON 结构，含文件路径、内容、描述、语言

### 3.2 AiProviderService.ts

- AI API 提供商的 CRUD 操作
- 对外隐藏 `apiKey` 敏感字段
- 支持启用/禁用状态管理

### 3.3 CodeInjector.ts

- 将 AI 生成的代码写入项目对应位置
- 支持自动建目录、自动备份（可回滚）
- 自动注册路由、自动写入菜单与权限资源

### 3.4 LocalFileRAG.ts

- 本地知识库检索引擎
- 基于关键词 + 路径权重 + 内容相似度
- 知识库来源：`AI_DEVELOPMENT_GUIDE.md` 和 `.ai-knowledge/`

## 四、AI 知识数据库（.ai-knowledge/）

知识库供 AI 检索，包含三类文件：

1. **project-specs/**：项目规范文档（代码规范、API 规范、数据库规范、前端组件规范、目录结构规范、AI 助手规范、SSE 与性能监控规范、安全加固规范）
2. **code-templates/**：代码模板（后端 Controller/Model/Service，前端 API/Page）
3. **reference-modules/**：参考示例模块（用户管理）

## 五、AI 助手使用流程

1. 用户点击右下角浮动按钮，打开抽屉
2. 在输入框输入需求，点击发送
3. AI 调用后端 `aiController` → `AIAssistant` → `AiProviderService`（选模型）+ `LocalFileRAG`（检索知识库）
4. AI 返回解释文本 + 生成的代码文件列表
5. 前端渲染 Markdown + 代码高亮，显示文件列表
6. 用户可逐文件查看、复制，或批量应用（写入文件 + 创建菜单）
7. 用户可在抽屉内管理 AI 提供商（添加/编辑/删除/启用/禁用）
