### 各层职责说明

- **config/**：环境配置、第三方服务连接配置（数据库、Redis 等）
- **models/**：数据库模型定义，字段映射，类型声明，实例方法
- **services/**：封装业务逻辑，一个功能模块对应一个 service，导出命名函数
- **controllers/**：只处理 HTTP 请求解析和响应输出，不包含业务逻辑，调用 service 处理
- **routes/**：定义路由路径、绑定中间件和控制器，添加 OpenAPI 注释
- **middleware/**：可复用的 Express 中间件（认证、日志、错误处理、限流等）
- **shared/schemas/**：所有接口参数的 Zod 校验 schema 定义，统一在这里
- **utils/**：通用工具函数，不依赖业务逻辑

## 导入路径规范

### 后端（TypeScript）

导入其他文件必须加上 `.js` 扩展名：
```typescript
// ✅ 正确
import User from '../models/User.js'
import { authMiddleware } from '../middleware/auth.js'

// ❌ 错误
import User from '../models/User'
import { authMiddleware } from '../middleware/auth'
```

### 前端（Vue/TypeScript）

使用别名 `@/` 指向 `src/`：
```typescript
// ✅ 正确
import request from '@/utils/request'
import type { User } from '@/types/api'

// ❌ 错误
import request from '../../utils/request.ts'
```

## 文件命名规范

### 后端
- 所有 TypeScript 文件：kebab-case + `.ts` 扩展名
  ✅ `user-controller.ts`、`user-service.ts`
  但是项目中实际使用：`userController.ts`、`userService.ts`（camelCase）

### 前端
- Vue 单文件组件：PascalCase + `.vue` 扩展名
  ✅ `UserList.vue`、`UserFormDialog.vue`
- TypeScript 文件：kebab-case + `.ts` 扩展名
  ✅ `user.ts`、`use-crud.ts`

## 模块边界

### 分层原则
- 依赖方向：Controller → Service → Model → DB
- Controller 不直接操作数据库，必须通过 Service
- Service 不关心 HTTP 请求响应，只处理业务逻辑
- 反向依赖禁止：Service 不依赖 Controller

### 前端模块边界
- 页面组件可以依赖组件和 API
- 组件尽量不依赖页面，可以依赖工具和类型
- 工具函数不依赖任何业务代码

## 新增功能流程

### 新增一个功能模块（知识库管理）

1. **后端**：
   - 创建 `server/migrations/..._create_knowledge_base.cjs`
   - 创建 `server/models/KnowledgeBase.ts`
   - 创建 `server/services/knowledgeBaseService.ts`
   - 创建 `server/controllers/knowledgeBaseController.ts`
   - 创建 `server/routes/knowledgeBaseRoutes.ts`
   - 在 `server/shared/schemas/knowledgeBase.ts` 添加校验 schema
   - 在 `server/routes/index.ts` 注册路由

2. **前端**：
   - 创建 `src/api/knowledge-base.ts` 接口定义
   - 创建 `src/views/knowledge-base/KnowledgeBaseList.vue` 页面
   - 在 `src/router/dynamicRoutes.ts` 添加路由配置
   - 在数据库插入菜单记录
