# 09-本地环境搭建

## 适用范围

重新 clone、换机器、MySQL 自动建库失败、迁移/种子失败时的本地搭建规范。完整可执行 skill：`.agents/skills/vue-admin-env-setup/`。

## 启动链路

```
server/.env
  → ensureDatabaseExists()   # query + CREATE DATABASE IF NOT EXISTS
  → sequelize.authenticate()
  → Umzug migrator.up()      # server/migrations
  → seeder.up() if users=0   # server/seeders
  → HTTPS listen SERVER_PORT
```

## 最短命令

```bash
npm run install:all
# 编辑 server/.env：DB_HOST/DB_PORT/DB_NAME/DB_USER/DB_PASSWORD/SERVER_PORT
npm run dev
```

手动迁移：

```bash
npm run migrate:status
npm run migrate
npm run migrate:seed
```

默认账号：`admin` / `123456`  
API 前缀：`/api/v1`  
健康检查：`/health`

## Windows 必知

1. 根 `package.json` 的 migrate 必须是 `npm --prefix server run migrate*`
2. `server/utils/migrator.ts` 的 glob 路径必须 `.replace(/\\/g, '/')`
3. 建库用 `query`，不要用 `execute(CREATE DATABASE)`
4. admin seeder 必须运行时 `bcrypt.hash`，禁止写死假哈希
5. `tsx`/`esbuild` 异常时可用 `tsc` + `node dist/app.js` 兜底

## 隔离验证

测试时换库名/端口，例如：

```ini
DB_NAME=vue_admin_skill_test
DB_PORT=3307
SERVER_PORT=5180
```

不要覆盖现有开发库。
