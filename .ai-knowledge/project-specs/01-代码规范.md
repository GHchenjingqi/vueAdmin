# 代码规范

## 后端（TypeScript + Express + Sequelize）

### 文件结构
- **models/** - Sequelize 数据模型定义
- **controllers/** - HTTP 请求/响应编排，只负责参数处理和响应
- **services/** - 业务逻辑封装，所有业务逻辑必须放在这里
- **routes/** - 路由定义，注册接口路径和处理器
- **middleware/** - Express 中间件
- **config/** - 配置文件
- **utils/** - 工具函数
- **shared/schemas/** - Zod 验证 schema 定义

### 编码风格
- 使用 **TypeScript**，充分利用类型系统
- 使用 **ESM 模块系统** (`import`/`export`，不是 `require`)
- 文件扩展名必须使用 `.js`（编译后仍然需要），导入时也要加 `.js`
  ```typescript
  // 正确
  import User from '../models/User.js'
  // 错误
  import User from '../models/User'
  ```
- 使用 `async/await` 处理异步，不使用回调
- Controller 中必须使用 `try/catch`，错误交给 `next(err)`
- 使用 JSDoc 注释每个导出函数和类
- 缩进：2 空格
- 字符串使用单引号
- 大括号不换行（K&R 风格）

### Model 规范
- 继承 `Model<InferAttributes<Model>, InferCreationAttributes<Model>>`
- 使用 `CreationOptional<T>` 标记自动生成字段（id, createdAt, updatedAt, status 默认值等）
- 使用 `DataTypes` 定义字段类型
- 必须添加 `comment` 描述
- 实例方法放在类定义内部
- 关联关系在模型定义后设置

示例：
```typescript
import { DataTypes, Model, CreationOptional, InferAttributes, InferCreationAttributes } from 'sequelize'
import sequelize from '../config/database.js'

class ModelName extends Model<InferAttributes<ModelName>, InferCreationAttributes<ModelName>> {
  declare id: CreationOptional<number>
  declare fieldName: string
  declare readonly createdAt: CreationOptional<Date>
  declare readonly updatedAt: CreationOptional<Date>
}

ModelName.init({
  id: {
    type: DataTypes.INTEGER.UNSIGNED,
    primaryKey: true,
    autoIncrement: true,
  },
  fieldName: {
    type: DataTypes.STRING(100),
    allowNull: false,
    comment: '字段描述',
  }
}, {
  sequelize,
  tableName: 'table_name',
})
```

### Controller 规范
- 职责：仅做 HTTP 请求/响应编排，业务逻辑委托给 Service
- 每个函数处理一个路由
- 使用 `try/catch` 包裹，错误传递给 `next(err)`
- 成功响应格式：`res.json({ code: 0, data: result, message: '成功' })`
- 创建使用 `res.status(201)`
- 操作日志：调用 `logOperation(req, '操作名', '描述', detailJson)`

### Service 规范
- 职责：封装所有业务逻辑
- 导出命名函数，不使用默认导出
- 依赖通过导入获取，不通过构造函数注入
- 使用 `AppError` 抛出业务错误

### 路由规范
- 所有路由注册到 `server/routes/index.ts`
- 使用 JSDoc + OpenAPI 注释生成 API 文档
- 需要认证的路由使用 `authMiddleware`
- 需要验证的路由使用 `validate(schema)`

### 错误处理
- 使用 `AppError` 抛出业务错误
- HTTP 状态码正确使用：
  - 400 - 请求参数错误
  - 401 - 未认证
  - 403 - 无权限
  - 404 - 资源不存在
  - 500 - 服务器错误

## 前端（Vue 3 + TypeScript + Element Plus）

### 文件结构
- **src/api/** - API 接口定义
- **src/components/** - 可复用组件
- **src/views/** - 页面组件
- **src/composables/** - 组合式函数
- **src/stores/** - Pinia 状态管理
- **src/router/** - 路由配置
- **src/types/** - TypeScript 类型定义
- **src/utils/** - 工具函数
- **src/i18n/** - 国际化

### 编码风格
- 使用 **Composition API** (`setup` 或 `<script setup>`)
- `<script setup>` 语法糖优先
- 使用 TypeScript，定义接口类型
- 缩进：2 空格
- 使用双引号
- 模板使用双引号，属性换行对齐

### 页面规范
- 使用 ProTable 组件快速构建 CRUD 页面
- 使用 i18n 国际化 `t()` 函数处理所有文字
- 表格列使用 `const columns = computed(() => [...])` 定义
- 搜索字段使用 `const searchFields = [...)]` 定义
- 使用 Element Plus 组件库
- 操作按钮放在 `template #header-buttons` 和 `template #column-actions` 中

示例结构：
```vue
<template>
  <div class="page-container">
    <ProTable ...>
      <template #header-buttons>
        <!-- 操作按钮 -->
      </template>
      <template #column-xxx="{ row }">
        <!-- 列自定义渲染 -->
      </template>
      <template #column-actions="{ row }">
        <!-- 行操作按钮 -->
      </template>
    </ProTable>
  </div>
</template>

<script setup lang="ts">
import { ref, computed } from 'vue'
import { useI18n } from 'vue-i18n'
import { Plus, Edit, Delete } from '@element-plus/icons-vue'
import ProTable from '@/components/ProTable/index.vue'
import { xxxApi } from '@/api/xxx'
// ...
</script>
```

### API 规范
- 在 `src/api/` 下定义接口，使用 `request` 封装
- 定义查询参数接口和返回类型
- 导出 `xxxApi` 对象，包含所有方法
- 使用泛型指定返回类型

## 数据库规范

### 表名
- 使用下划线命名法（snake_case）
- 使用复数形式（如 `users`, `role_menus`）
- 多词表名：`user_roles`, `dict_data`

### 字段名
- 使用下划线命名法
- 必须使用 `id` 作为主键
- 外键格式：`{table}_id`（如 `user_id`, `role_id`）
- 布尔字段：使用 `TINYINT(1)`，`0`/`1`，默认值 `0` 或 `1`
- 时间字段：使用 `createdAt`, `updatedAt`（Sequelize 自动维护）
- 必填字段：`allowNull: false`
- 所有字段必须添加 `comment` 描述

### 字符集
- 数据库/表字符集：`utf8mb4`
- 排序规则：`utf8mb4_unicode_ci`

## 通用规范

### 命名约定
- **文件**：kebab-case（如 `userController.ts`, `UserList.vue`）
- **类**：PascalCase（如 `User extends Model`）
- **函数/变量**：camelCase
- **常量**：UPPER_SNAKE_CASE
- **类型/接口**：PascalCase

### Git 提交
- 遵循 Conventional Commits 规范
- `feat:` 新功能
- `fix:` 修复
- `docs:` 文档
- `refactor:` 重构
- `perf:` 性能优化
- `test:` 测试
- `chore:` 构建/工具

### 导入顺序
1. Node.js 内置模块
2. 第三方依赖
3. 内部模块（按相对路径从短到长）
4. 样式/资源

每个分组之间空一行。
